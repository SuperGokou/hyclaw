# HYClaw V1 验收标准

- 版本:1.0(2026-07-16)
- 适用:HYClaw V1 全部里程碑(M1–M6)
- 用法:每个里程碑交付后,按对应章节逐条操作验证;**全部 ☐ 勾选通过才算里程碑验收通过**。V1 最终验收 = M1–M6 全部通过 + 第 7 节整体验收全部通过。

## 1. M1 验收 — Fork + 品牌化 + 桌面壳

### 1.1 品牌化

- ☐ 启动应用,窗口标题为 **HYClaw · 和熠智脑**,且切换页面时标题不变回 OpenClaw
- ☐ 任务栏图标、窗口图标为 **EFD logo**(红色 EFD 字样)
- ☐ 控制台左侧边栏品牌字为 **HYClaw**;顶栏品牌字为 **HYClaw**
- ☐ 「关于」页产品名显示 **HYClaw**
- ☐ 启动过程首先出现品牌加载页(EFD logo + "正在启动本地智能体网关…")

### 1.2 桌面壳行为

- ☐ 启动后 60 秒内自动进入控制台界面,无需手动刷新
- ☐ 在对话框发一条消息,能收到 AI 回复(前提:已配置任一模型 API key)
- ☐ 点窗口右上角 ✕:窗口隐藏,应用**不退出**,系统托盘图标仍在
- ☐ 双击托盘图标:主窗口恢复显示
- ☐ 托盘右键菜单有「打开 HYClaw / Open」「退出 / Quit」两项,中英双语
- ☐ 托盘「退出」后:在 PowerShell 运行 `netstat -ano | findstr 18789`,**无输出**(网关进程已被正确关闭,无残留)
- ☐ 应用运行中再次启动 HYClaw:不出现第二个窗口,而是聚焦已有窗口(单实例)
- ☐ 强杀网关进程(任务管理器结束 node 进程):应用自动重启网关,窗口恢复可用;连杀 4 次后显示故障页而不是无限重启

### 1.3 工程质量

- ☐ `pnpm --filter @hyclaw/desktop test` 全绿
- ☐ `pnpm --filter openclaw-control-ui exec vitest run src/pages/about/view.test.ts` 全绿
- ☐ `pnpm ui:build` 与 `pnpm build` 成功
- ☐ 代码仓库已推送 `https://github.com/SuperGokou/hyclaw`,`git remote -v` 中 origin=SuperGokou/hyclaw、upstream=openclaw(仅 fetch)
- ☐ `apps/desktop/release/win-unpacked/HYClaw.exe` 存在且可启动(设置 `HYCLAW_REPO_ROOT` 后)

## 2. M2 验收 — HYShield 安全围栏 + 模型配置

### 2.1 网络围栏

- ☐ 应用运行中,`netstat -ano | findstr 18789` 显示监听地址为 **127.0.0.1**,绝不是 0.0.0.0
- ☐ 用局域网内另一台电脑访问 `http://<本机IP>:18789/`:**连接被拒绝**
- ☐ 手工把网关配置的 bind 改为 0.0.0.0 再启动:应用**拒绝启动**并给出中文错误说明(HYShield 自检)

### 2.2 凭证保护

- ☐ 在设置页录入一个模型 API key 并保存
- ☐ 用文本编辑器打开用户目录下所有 HYClaw/openclaw 配置文件,**搜索不到该 key 的明文**(全盘搜索关键字也搜不到)
- ☐ 重启应用后该 key 仍可用(加密持久化生效)
- ☐ 未通过认证的 WebSocket 客户端连接 18789 端口:被拒绝

### 2.3 出站白名单与审计

- ☐ 白名单只配置模型 API 域名时,让 Agent 访问一个不在白名单的网址:请求被拦截,并有明确提示
- ☐ 审计日志文件存在,包含:出站请求记录、凭证使用记录、skill 加载记录;日志中**不出现任何明文密钥**
- ☐ 非白名单 skill 无法被加载(手工放一个未登记的 skill 目录验证)

### 2.4 模型配置页

- ☐ 三类 Provider 均可配置:Anthropic 官方 / OpenAI 兼容端点(填 base URL)/ Ollama 本地
- ☐ 每类配置有「连通性测试」按钮,成功/失败均有明确反馈
- ☐ 配置 DeepSeek 或通义等 OpenAI 兼容端点后,对话可用
- ☐(如有本地 Ollama)选择本地模型后,断网状态下对话仍可用

## 3. M3 验收 — Office skills + 数据可视化 + 双语

### 3.1 Office 文档能力

准备一个测试工作区文件夹,放入样例 xlsx/docx/pptx/pdf 各一份:

- ☐ **Excel 读**:让 Agent 总结 xlsx 内容,回答与实际数据一致
- ☐ **Word 写**:让 Agent 根据 Excel 数据生成一份 Word 报告,文件可用 Office/WPS 正常打开,格式无乱码
- ☐ **PPT 生成**:让 Agent 生成 5 页以内的简报 pptx,可正常打开放映
- ☐ **PDF 读**:让 Agent 提取 PDF 要点,内容正确
- ☐ **路径围栏**:让 Agent 读工作区外的文件(如 C:\Windows\ 下任意文件):被拒绝

### 3.2 数据可视化

- ☐ 控制台存在「数据可视化」页签
- ☐ 让 Agent 分析样例 Excel 并出图:图表在面板中渲染(柱/折/饼至少各验证一次)
- ☐ 图表可导出 PNG,文件可打开
- ☐ 生成的 Word/PPT 报告中可嵌入该图表

### 3.3 中英双语

- ☐ 设置中可切换 中文/English,切换后界面主要区域(侧边栏、设置、对话页)全部跟随
- ☐ 默认首次启动为中文
- ☐ 中文界面下无明显英文残留(允许专有名词);英文界面下无中文残留
- ☐ 中文提问得到通顺中文回答;英文提问得到英文回答

## 4. M4 验收 — IM 渠道 + MCP + 桌面伴侣

### 4.1 企业微信 / 钉钉

- ☐ 配置页可录入企微自建应用凭证,凭证保存后同 2.2 一样查无明文
- ☐ 手机企业微信 @机器人发消息,收到 Agent 回复(即"IM 即手机端")
- ☐ 钉钉同上
- ☐ 陌生人/非白名单成员发消息:Agent **不处理**(注入防护默认策略)
- ☐ 断网重连后渠道自动恢复,无需重启应用

### 4.2 通用 MCP

- ☐ 设置页可添加一个 MCP server(以任一公开 MCP server 为例),连接状态可见
- ☐ Agent 能调用该 MCP server 提供的工具完成一次任务

### 4.3 桌面伴侣 HYPet

- ☐ 桌宠悬浮窗置顶显示,可拖动,可收起
- ☐ 任务完成时有文字气泡提示;开启语音后有中文语音播报
- ☐ 全局热键可唤起快捷对话
- ☐ 桌宠能感知前台应用(在 Excel 前台时给出情境化入口提示)
- ☐ 设置中可完全关闭桌宠与语音;关闭后无任何残留提示
- ☐ **隐私边界**:桌宠不做持续截屏/录屏(V1 承诺);仅用户主动触发截屏求助时才读屏,且触发时有明显视觉指示

## 5. M5 验收 — 智能体空间(3D)+ 手机局域网配对

### 5.1 智能体空间

- ☐ 控制台存在「智能体空间」页签,打开后进入 3D 虚拟办公室
- ☐ 3D 场景中能看到当前 Agent 会话的实时活动
- ☐ 场景内品牌元素为 HYClaw/EFD(无 Claw3D 原品牌残留)
- ☐ 低配开关:设置中关闭 3D 后,页签隐藏,主功能不受影响
- ☐ 3D 页签未打开时不占用显著 GPU/内存(任务管理器对比验证)

### 5.2 手机局域网配对

- ☐ 配对功能**默认关闭**
- ☐ 开启后桌面端显示二维码;同一 Wi-Fi 手机扫码后浏览器打开 HYClaw 控制台,可对话
- ☐ 配对令牌过期后(按配置时长),手机端要求重新配对
- ☐ 不同 Wi-Fi/热点下的设备无法访问(仅内网网段)
- ☐ 手机已连接时,桌面端有常驻「手机已连接」指示
- ☐ 关闭配对功能后,手机端立即断开,`netstat` 确认端口回到仅 127.0.0.1
